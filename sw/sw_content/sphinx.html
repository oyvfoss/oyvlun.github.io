Tables¶

Fairly simple, graphical syntax. Pluses are corners, dashes are horizontal lines, |’s are vertical lines. = for headers. The table style depends on html configuration. Currently this does not display column dividers. I should look into the Cloud Sphinx theme extension, which adds more options for tables.

+------------+------------+-----------+
| Header 1   | Header 2   | Header 3  |
+============+============+===========+
| body row 1 | column 2   | column 3  |
+------------+------------+-----------+
| body row 2 | Cells may span columns.|
+------------+------------+-----------+
| body row 3 | Cells may  | - Cells   |
+------------+ span rows. | - contain |

Admonitions¶

Specific admonitions (attention, caution, danger, error, hint, important, note, tip, warning).

.. warning::
   This is some warningful text.

.. note::
   This is some hintful text.

Yields:

Warning

This is some warning text.

Note

This is some noteful text.

Generic admonitions where the header text is specified are written like this:

.. admonition:: My headline

   This is my admonition.

Yielding:

My headline

This is my admonition.

Theme customizations¶

The most straight-forward way to customize the theme is to make local changes in conf.py.

For example, I’ve specified the following to change background colors for adminition boxes in the alabaster theme:

html_theme_options = {
   'admonition_bg':'#ADD8E6',
   'note_bg':'#90EE90',

}
