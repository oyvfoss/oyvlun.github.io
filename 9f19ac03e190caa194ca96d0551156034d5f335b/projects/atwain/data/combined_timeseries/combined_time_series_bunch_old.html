All time series data colleted in common file (old version)¶

Warning

Redoing this in early January 2020. This is the previous version, from something like October-December 2019. Keeping the page (and the scripts= for reference.

Collecting all variables in one single file

atwain/data/atwain_timeseries_collection_old.p

This file is produced in processing_and_data/collect_time_series/create_common_atwain_bunch_old.py.

  • Creating a Bunch object for storing the file.

  • Collecting all variables (sources specified below).

  • For HF data (e.g. mooring data): Smooth with a daily average running mean.

  • Interpolate time series onto a common, daily time grid.

  • For each variable:

    • Units are stored in units[varname].

    • Short description is stored in name[varname].

    • Longer description is stored in longname[varname].

    • Data source stored in data_source[varname].

Time¶

Time grid is daily from 01.08.2012 to 31.12.2017. The following time fields are included in the Bunch:

  • t: Time, matplotlib format.

  • t_matlab: Time, matlab format (t+366).

  • yr: Year

  • mo: Month

  • yday: Yearday

✕ “Preliminary” status ✕

T50, T105, dsig:

These are computed from the interpolated seabird data post 2013. Should check exactly how this interpolation was done. Also: T105 pre 2013 is just the temperature record from the sensor. The depths are pretty close to 105 always, though.

Also, there are some artefacts around deployment times. Need to check that deployment and recovery have in fact been chopped off.

None of these things are critical for what I do here - should be fine for the time being.

airT, wsp, wdir, t_storm, t_freeze

Spatial interpolation based on RectBivariateSplite - should use the xesmf Regridder. Shouldn’t matter much.

SST

Currently getting from my previously extracted SST for the A-TWAIN point. Evcentually want to interpolate using xesmf and do an average for the sea ice area.

Current direction

u_al, u_cr:

Rotating such that u_al is directed towards 14.5⁰ - the approximate direction estimated from the bathymetry.

Can be modified (in degrees) to ang_rot_deg.
